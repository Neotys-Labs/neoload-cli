# External Integrations

**Analysis Date:** 2026-04-08

## APIs & External Services

**NeoLoad Web REST API (Primary Integration):**
- NeoLoad Web (SaaS or self-hosted) - All test orchestration: run tests, read results, manage settings, zones, workspaces
  - SDK/Client: `requests` + custom `rest_crud.py` wrapper at `neoload/neoload_cli_lib/rest_crud.py`
  - Auth: Bearer token passed as `accountToken` HTTP header; stored in `config.yaml` after `neoload login`
  - Default SaaS URL: `https://neoload-api.saas.neotys.com/`
  - API versions: `v2`, `v3`, `v3/workspaces/{workspaceId}` (version selected based on NLWeb server version)
  - Endpoints used: `/test-results`, `/test-settings`, `/resources/zones`, `/workspaces`, `/information`, `explore/v2/swagger.yaml`, `/nlweb/rest/rest-api/url-api/v1/action/get-front-end-url`
  - Retry logic: 10 retries on HTTP 429 with server-specified `Retry-After` delay (milliseconds-based) via `urllib3.util.retry.Retry`
  - File upload endpoint: separate `file_storage_url` obtained from `/v3/information` or Swagger spec

**NeoLoad Web File Storage API:**
- Separate base URL from the main API (resolved on login from `/v3/information` or Swagger)
- Used for project uploads: `neoload/neoload_cli_lib/rest_crud.py` (`post_binary_files_storage`)
- Multipart upload with `requests_toolbelt.MultipartEncoder` and `tqdm` progress bar

**GitHub Raw Content:**
- NeoLoad YAML project schema fetched from: `https://raw.githubusercontent.com/Neotys-Labs/neoload-models/v3/neoload-project/src/main/resources/as-code.latest.schema.json`
- Used by `neoload/neoload_cli_lib/schema_validation.py` to validate project YAML against JSON Schema Draft7
- Schema is cached locally; only re-fetched when remote differs (SHA256 hash comparison)

## Data Storage

**Databases:**
- None - no database. All persistent state in YAML files on local filesystem.

**File Storage:**
- Local filesystem config files (platform-specific via `appdirs`):
  - `config.yaml` - Login credentials (token, URL, workspace, metadata) at `appdirs.user_data_dir("neoload-cli", "neotys", "1.0")/config.yaml`
  - `.neoload_cli.yaml` - Local project-level config override (if present in cwd)
  - `config-global.yaml` - Global CLI settings (Docker config, etc.) at same `appdirs` directory
  - `yaml_schema.json` - Cached NeoLoad YAML schema at same `appdirs` directory
- Code: `neoload/neoload_cli_lib/user_data.py`, `neoload/neoload_cli_lib/config_global.py`, `neoload/neoload_cli_lib/paths.py`

**Caching:**
- JSON schema cached to disk (see File Storage above)
- No in-memory or distributed cache

## Authentication & Identity

**Auth Provider:**
- NeoLoad Web token-based authentication (not OAuth; custom `accountToken` header)
  - Implementation: Token stored in local YAML config file after `neoload login <token> --url <url>`
  - Token masked in display output (all but last 3 characters replaced with `*`)
  - SSL cert verification configurable: path to cert file, `False` to disable, or default `True`
  - See `neoload/neoload_cli_lib/user_data.py` and `neoload/commands/login.py`

## Monitoring & Observability

**Error Tracking:**
- None embedded in the CLI itself

**Code Quality / Coverage:**
- SonarCloud - Static analysis and coverage reporting
  - Config: `sonar-project.properties` (org: `neotys`, project: `neoload-cli`)
  - Coverage input: `coverage.xml` generated by `coverage3 xml`
  - Integrated in CI via `SonarSource/sonarqube-scan-action@v6.0.0` (GitHub Actions) and `sonarcloud` addon (Travis CI)
  - Secrets: `SONAR_TOKEN` (GitHub secret)

**Logs:**
- Python `logging` module with `coloredlogs` for terminal output
- Log level controlled by `--debug` flag on the root `cli` command (`neoload/__main__.py`)
- `urllib3` and `requests` debug logging enabled when `--debug` is active

## CI/CD & Deployment

**Hosting:**
- PyPI - Package published as `neoload`
  - Release workflow: `.github/workflows/python-release.yml`
  - Publish action: `pypa/gh-action-pypi-publish`
  - Secret: `PYPI_TOKEN`
  - Triggered on git tags (`refs/tags/**`)

**CI Pipeline:**
- GitHub Actions (primary):
  - `.github/workflows/python-package.yml` - Unit tests + optional integration tests + SonarCloud; runs on push/PR; matrix: Python 3.8, 3.11
  - `.github/workflows/python-integration-tests.yml` - Integration tests
  - `.github/workflows/python-release.yml` - Build and publish to PyPI on tags
  - `.github/workflows/python-post-release-changelog-pr.yml` - Post-release automation
- Travis CI (legacy):
  - `.travis.yml` - Python 3.6 and 3.8; unit + live call tests; deploys to PyPI on tags; SonarCloud scan

## Docker Integration

**Docker SDK:**
- `docker` Python SDK used directly (`neoload/neoload_cli_lib/docker_lib.py`)
- Connects to local Docker daemon via `docker.from_env()`
- Manages NeoLoad controller and load generator containers:
  - Default controller image: `neotys/neoload-controller:latest`
  - Default load generator image: `neotys/neoload-loadgenerator:latest`
  - Images pulled from Docker Hub if not present locally
- Containers labeled `launched-by-neoload-cli` for identification
- Container lifecycle managed via `neoload docker up/down/clean/kill` commands and test lifecycle hooks

## Webhooks & Callbacks

**Incoming:**
- None - the CLI is a client-only tool

**Outgoing:**
- None in the webhook sense; the CLI polls NeoLoad Web API for test status during `neoload run` (1-second polling interval in `neoload/neoload_cli_lib/running_tools.py`)

## Environment Configuration

**Required env vars (for live/integration testing):**
- `NLWEB_TOKEN` (or `--token` CLI arg) - NeoLoad Web API token
- `NLWEB_API_URL` (or `--url` CLI arg) - NeoLoad Web API base URL

**Optional env vars:**
- `NL_INTERACTIVE` - Force interactive mode even in non-TTY environments
- `COLORTERM`, `TERM_PROGRAM`, `TERM` - Terminal color detection
- CI detection vars (auto-detected, no manual config needed): `JENKINS_URL`, `TRAVIS`, `bamboo_buildNumber`, `CODEBUILD_BUILD_ARN`, `AGENT_TOOLSDIRECTORY`, `CI_PROJECT_ID`, `TEAMCITY_VERSION`, `CIRCLECI`, `BUILD_ID`, `CI`, `CONTINUOUS_INTEGRATION`

**Secrets location:**
- GitHub Actions secrets: `NLWEB_TOKEN`, `NLWEB_API_URL`, `SONAR_TOKEN`, `GITHUB_TOKEN`, `PYPI_TOKEN`
- Travis CI: encrypted token in `.travis.yml` for SonarCloud and PyPI
