# Phase 6: Infrastructure

v4 commands for proxies, infrastructure providers, and guaranteed resources.

## Tasks

### 1. v4-proxies (`v4_proxies.py`)

Subcommands:
- `ls` — GET /v4/proxies
- `create` — POST /v4/proxies
- `patch` — PATCH /v4/proxies/{proxyId}
- `delete` — DELETE /v4/proxies/{proxyId}

No workspaceId needed on any operation.

### 2. v4-infrastructure-providers (`v4_infrastructure_providers.py`)

Subcommands:
- `ls` — GET /v4/infrastructure-providers
- `create` — POST /v4/infrastructure-providers
- `patch` — PATCH /v4/infrastructure-providers/{id}
- `delete` — DELETE /v4/infrastructure-providers/{id}

No workspaceId needed on any operation.

### 3. v4-guaranteed-resources (`v4_guaranteed_resources.py`)

Subcommands:
- `ls` — GET /v4/workspaces/{workspaceId}/guaranteed-resources
- `create` — POST /v4/workspaces/{workspaceId}/guaranteed-resources
- `patch` — PATCH /v4/workspaces/{workspaceId}/guaranteed-resources
- `delete` — DELETE /v4/workspaces/{workspaceId}/guaranteed-resources

All operations require workspaceId as a **path parameter** (not query/body). Uses stored workspace ID or `--workspace` option.
