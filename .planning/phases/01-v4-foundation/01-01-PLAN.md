# Phase 1: v4 API Foundation

Create the shared v4 helper package that all v4 commands will depend on.

## Objective

Build `neoload/neoload_cli_lib/v4/` as a new package with path construction helpers and convenience HTTP wrappers that delegate to the existing `rest_crud` module.

## Tasks

### 1. Create v4 package — `__init__.py`

Create `neoload/neoload_cli_lib/v4/__init__.py` as a package marker.

### 2. Create `v4_endpoints.py`

Path construction helpers:

- `v4_base()` — returns `"v4"`
- `v4_endpoint(resource, resource_id=None)` — builds `"v4/{resource}"` or `"v4/{resource}/{id}"`
- `v4_workspace_params()` — returns `{"workspaceId": id}` dict from stored workspace; raises `CliException` if no workspace stored. Opt-in only.
- `v4_inject_workspace(data)` — returns a copy of `data` with `workspaceId` added from stored workspace; raises `CliException` if no workspace stored. Opt-in only.

Reads workspace from `user_data.get_meta('workspace id')` — the same value set by existing `login --workspace` / `workspaces use` commands.

### 3. Create `v4_client.py`

Thin convenience wrappers combining `rest_crud` functions with v4 path building:

- `v4_list(resource, params=None)` — GET with pagination, caller passes workspace params if needed
- `v4_get(resource, resource_id)` — GET single resource by ID
- `v4_create(resource, data)` — POST, caller injects workspace into data if needed
- `v4_update(resource, resource_id, data)` — PATCH
- `v4_replace(resource, resource_id, data)` — PUT
- `v4_delete(resource, resource_id)` — DELETE

All functions import and call `rest_crud.get`, `rest_crud.post`, etc. No HTTP logic is duplicated.

### 4. Unit tests

- `tests/neoload_cli_lib/v4/test_v4_endpoints.py` — test path building, workspace param injection, error on missing workspace
- `tests/neoload_cli_lib/v4/test_v4_client.py` — test that client functions call rest_crud with correct paths and parameters
