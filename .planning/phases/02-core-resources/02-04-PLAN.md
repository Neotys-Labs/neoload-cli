CRITICAL DETAILS:
- Workspaces are NOT workspace-scoped. Do NOT use v4_list or v4_create.
- ls: use rest_crud.get with pageNumber/pageSize pagination (manual loop, same as zone ls)
- create: use rest_crud.post directly (NOT v4_create which injects workspaceId into body)

- members-remove: DELETE with ?login=X query param. rest_crud.delete has no params arg. Use urllib.parse.urlencode to encode login in URL.
- Workspace ID format is MongoDB ObjectId (24 hex chars), NOT UUID. Use tools.is_mongodb_id() for validation.

5. `_workspace_list` helper (manual pagination — NOT v4_list):
```python
def _workspace_list():
    """List all workspaces. Does NOT use v4_list (which injects workspaceId)."""
    page_number = 0
    page_size = 200
    all_items = []
    while True:
        response = rest_crud.get(
            v4_endpoints.v4_endpoint('workspaces'),
            {'pageNumber': page_number, 'pageSize': page_size}
        )
        items = response.get('items', [])
        all_items.extend(items)
        total = response.get('total', 0)
        if len(all_items) >= total or not items:
            break
        page_number += 1
    return all_items
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> URL path | Workspace ID and member login values become URL path/query segments |
| --file -> json.load | User-provided JSON for create, members-add |
| --login -> URL query | User login value URL-encoded into DELETE query string |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Tampering | `--file` JSON input | mitigate | `json.load()` validates JSON; malformed raises CliException |
| T-02-12 | Tampering | `--login` in members-remove URL | mitigate | `urllib.parse.urlencode({'login': login})` safely encodes special chars (@, spaces, &) preventing URL injection |
| T-02-13 | Tampering | workspace `id` in URL path | accept | ID passed via v4_endpoint path join; API validates server-side |
| T-02-14 | Elevation of Privilege | members-add role assignment | accept | API-side authorization enforcement; CLI passes role value from user input |
</threat_model>
