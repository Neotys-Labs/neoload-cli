```python
def _build_body(input_file, name=None, description=None, scenario=None):
    body = {}
    if input_file:
        try:
            body = json.load(input_file)
        except json.JSONDecodeError as err:
            raise cli_exception.CliException(
                '%s\nThis command requires valid JSON input.' % str(err))
    if name is not None:
        body['name'] = name
    if description is not None:
        body['description'] = description
    if scenario is not None:
        body['scenarioName'] = scenario
    return body
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> v4_client | User-provided IDs and flag values passed to API path/body construction |
| --file -> json.load | User-provided JSON file content parsed and sent to API |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `--file` JSON input | mitigate | `json.load()` validates JSON structure; malformed input raises `CliException` with parse error |
| T-02-02 | Tampering | `id` argument in URL path | accept | IDs are passed via `v4_endpoint()` path join; no shell/SQL injection vector in URL path segments; API validates ID format server-side |
| T-02-03 | Information Disclosure | API error responses printed to stdout | accept | CLI tool runs locally; error messages from API are informational, no credential leakage (accountToken is in headers, not logged) |
</threat_model>
