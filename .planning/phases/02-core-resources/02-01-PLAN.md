# Phase 2: Core Resources

v4 commands for the primary resource types: tests, results, test-executions, workspaces, zones.

## Tasks

### 1. v4-tests (`v4_tests.py`)

Endpoints: `/v4/tests`, `/v4/tests/{testId}`, `/v4/tests/{testId}/scenarios/{scenarioName}`

Subcommands:
- `ls` — GET /v4/tests (requires workspaceId query param)
- `create` — POST /v4/tests (requires workspaceId in body)
- `get` — GET /v4/tests/{testId}
- `patch` — PATCH /v4/tests/{testId} (workspaceId optional in body — moves test)
- `delete` — DELETE /v4/tests/{testId}
- `scenario-get` — GET /v4/tests/{testId}/scenarios/{scenarioName}
- `scenario-update` — PUT /v4/tests/{testId}/scenarios/{scenarioName}

Options: `--file` for JSON body input, `--workspace` override for specific operations.

### 2. v4-results (`v4_results.py`)

Endpoints: `/v4/results`, `/v4/results/{resultId}`, plus sub-resources

Subcommands:
- `ls` — GET /v4/results (requires workspaceId query param)
- `get` — GET /v4/results/{resultId}
- `patch` — PATCH /v4/results/{resultId}
- `delete` — DELETE /v4/results/{resultId}
- `contexts` — GET /v4/results/{resultId}/contexts
- `elements` — GET /v4/results/{resultId}/elements
- `monitors` — GET /v4/results/{resultId}/monitors
- `statistics` — GET /v4/results/{resultId}/statistics
- `timeseries` — GET /v4/results/{resultId}/timeseries
- `search-criteria` — GET /v4/results/search-criteria (requires workspaceId)

### 3. v4-test-executions (`v4_test_executions.py`)

Endpoints: `/v4/test-executions`, `/v4/test-executions/{testExecutionId}`, `/v4/results/{resultId}/logs`

Subcommands:
- `create` — POST /v4/test-executions (structured JSON body with testId, configuration)
- `get` — GET /v4/test-executions/{testExecutionId}
- `cancel` — DELETE /v4/test-executions/{testExecutionId}
- `force-cancel` — DELETE /v4/test-executions/{testExecutionId}/forced
- `logs` — GET /v4/results/{resultId}/logs

Options: `--test-id`, `--scenario`, `--zone-type`, `--duration`, `--file` for full JSON config.

### 4. v4-workspaces (`v4_workspaces.py`)

Endpoints: `/v4/workspaces`, `/v4/workspaces/{workspaceId}`, members, subscription

Subcommands:
- `ls` — GET /v4/workspaces
- `create` — POST /v4/workspaces
- `get` — GET /v4/workspaces/{workspaceId}
- `patch` — PATCH /v4/workspaces/{workspaceId}
- `delete` — DELETE /v4/workspaces/{workspaceId}
- `members-ls` — GET /v4/workspaces/{workspaceId}/members
- `members-add` — POST /v4/workspaces/{workspaceId}/members
- `members-remove` — DELETE /v4/workspaces/{workspaceId}/members
- `subscription` — GET /v4/workspaces/{workspaceId}/subscription

### 5. v4-zones (`v4_zones.py`)

Endpoints: `/v4/zones`, `/v4/zones/{zoneId}`

Subcommands:
- `ls` — GET /v4/zones (optional type filter: STATIC, DYNAMIC, CLOUD)
- `create` — POST /v4/zones
- `get` — GET /v4/zones/{zoneId}
- `patch` — PATCH /v4/zones/{zoneId}
- `delete` — DELETE /v4/zones/{zoneId}
