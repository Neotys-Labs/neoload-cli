# Phase 3: Analytics and Trends

v4 commands for result analytics (element/monitor data, intervals) and test trends.

## Tasks

### 1. v4-analytics (`v4_analytics.py`)

Subcommands:
- `element-values` — GET /v4/results/{resultId}/elements/values
- `element-timeseries` — GET /v4/results/{resultId}/elements/{elementId}/timeseries
- `element-percentiles` — GET /v4/results/{resultId}/elements/{elementId}/percentiles
- `monitor-values` — GET /v4/results/{resultId}/monitors/values
- `monitor-timeseries` — GET /v4/results/{resultId}/monitors/{monitorId}/timeseries
- `intervals-ls` — GET /v4/results/{resultId}/intervals
- `intervals-create` — POST /v4/results/{resultId}/intervals
- `intervals-patch` — PATCH /v4/results/{resultId}/intervals/{intervalId}
- `intervals-delete` — DELETE /v4/results/{resultId}/intervals/{intervalId}
- `interval-generation` — POST /v4/results/{resultId}/interval-generation
- `report` — POST /v4/results/{resultId}/report

Options: `--result-id` (required), `--element-id`, `--monitor-id`, `--interval-id`, `--file` for JSON body.

### 2. v4-trends (`v4_trends.py`)

Subcommands:
- `get` — GET /v4/tests/{testId}/trends
- `patch` — PATCH /v4/tests/{testId}/trends
- `config-get` — GET /v4/tests/{testId}/trends/configuration
- `config-put` — PUT /v4/tests/{testId}/trends/configuration (supports --dry-run query param)
- `config-patch` — PATCH /v4/tests/{testId}/trends/configuration
- `elements` — GET /v4/tests/{testId}/trends/elements

Options: `--test-id` (required), `--dry-run` flag for config-put.
